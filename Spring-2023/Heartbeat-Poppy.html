FLORA YANTING ZHOU

Heartbeat Poppy

Type:
Art Installation

Duration:
04.2023

Personal creative technology project. 

Course

Creative Technology 4

Specification

Physical Computing
Generative Art

Software

TouchDesigner
Arduino IDE

Hardware

Arduino Nano
MAX30102

Ideation

Visualizing the Concept with Midjourney

Prompt:
"A digital garden consist of virtual generative plants base on sensor data collected from people's behavior, 3D render"

Design Process

Step 1 - Hardware Input

The MAX30102 is an integrated pulse oximetry and heart-rate monitor biosensor module. 

I use a Arduino Nano board to connect the sensor, with the intention of making a light-weight wearable wristband.

To test the functionality of my hardware, I included the MAX 30102 sensor libraries, which provided example codes. I used the Hearbeat Plotter example on the right. 

Additionally, I downloaded MQTT, Adafruit IO, and Vuforia libraries with the intention of collecting and storing data from the sensor and accessing cloud data from Unity. 

However, I realized that I would need an IoT board such as the ESP32 to use the IO cloud. Since I did not have the board available, I ultimately decided to abandon the IoT idea and instead focus on the core concept of the project.

Although the heartbeat plotter example provided real-time string data from the sensor, I needed to count the actual number of heartbeats through the serial port. 

To achieve this, I found an example code online which included a buzzer to count the beats and display them. Incorporating the buzzer was a useful addition as it made the heartbeat more apparent. 

Furthermore, I noticed that the code used to activate the buzzer could be modified to keep a count in the serial port. To achieve this, I declared an integer variable called "beatNum" and incremented it each time a heartbeat was detected.

Step 2 - Software Graphics

In order to create graphics with both complex shapes and a natural growth feel, I sought out an instancing tutorial of TouchDesigner.

During the process, I encountered an issue with instancing where extra geometries were appearing at one point of the original geometry.

However, I was able to resolve this issue by changing the instance number from automatic to manual and reducing the number of instances.

Step 3 - Connect!

To integrate the heartbeat input with the graphics, I created a node tree to receive the serial data and organized the nodes into a container.

I utilized the count data to control three different parameters by mapping the count data to various ranges. 

Firstly, the count data was mapped to control the size of the tube's top, where a larger count number resulted in a wider tube bottom, simulating the opening of a flower. 

Secondly, the count data was mapped to control the phase of the color ramp, resulting in the color of the simulation changing as the count number increased. 

Lastly, a mapped copy of the count data was used to control the number of particles, which simulates the pollen.

Final Output

Sit down and have a meditation with the poppy, let it grow with your heartbeat...
